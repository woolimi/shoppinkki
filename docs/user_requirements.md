# 사용자 요구사항 (User Requirements)

> **프로젝트:** 쑈삥끼 (ShopPinkki)
> **팀:** 삥끼랩 | 에드인에듀 자율주행 프로젝트 2팀

---

## UR 테이블

| UR ID | 카테고리 | Description |
|---|---|---|
| UR-01 | 진입 | 고객은 쑈삥끼 LCD에 항상 표시되는 QR 코드를 스캔하여 웹앱에 접속할 수 있다. |
| UR-02 | 계정 | 웹앱 최초 접속 시 회원가입 또는 로그인을 한다. 계정 정보(ID, 비밀번호, 이름, 전화번호, 카드 정보)는 중앙 서버에 영구 보관되며, 어느 쑈삥끼에서든 동일한 계정으로 이용할 수 있다. |
| UR-03 | 계정 / 포즈 스캔 | 카트 사용 시작 시 로그인/회원가입 직후 포즈 스캔을 수행한다. 포즈 스캔은 매 사용마다 새로 진행한다. |
| UR-04 | 세션 | 세션이 유효한 상태에서 QR 코드를 재접속하면 로그인 및 포즈 스캔을 건너뛰고 메인화면으로 이동한다. |
| UR-05 | 추적 | 포즈 스캔이 완료되면 쑈삥끼는 주인을 인식하고 마트 안에서 졸졸 따라다니며 쇼핑을 보조한다. |
| UR-06 | 추적 / 재탐색 | 쑈삥끼가 주인을 놓쳤을 경우, 제자리에서 방향을 전환하며 주인을 재탐색한다. 재탐색에도 주인을 찾지 못하면 대기 모드로 전환한다. |
| UR-07 | 대기 / 직원 호출 | 대기 모드 중 일정 시간 동안 주인이 돌아오지 않으면 직원을 호출한다. 호출은 중앙 관제 대시보드 알람 및 로그, 쑈삥끼 LED 점멸, LCD 화면 표시로 이루어진다. |
| UR-08 | 앱 / 상태 확인 | 주인은 웹앱에서 마트 맵, 현재 쑈삥끼의 위치, 현재 동작 모드를 실시간으로 확인할 수 있다. |
| UR-09 | LCD / 상태 표시 | 쑈삥끼 LCD는 항상 현재 동작 모드와 웹앱 접속용 QR 코드를 함께 표시한다. (LCD는 터치 조작 불가 — 조작은 웹앱에서만 가능) |
| UR-10 | 앱 / 모드 전환 | 주인은 웹앱을 통해 쑈삥끼를 대기 모드로 전환하거나, 대기 모드를 해제하고 다시 따라오기 모드로 복귀시킬 수 있다. (화장실 등 자리 비움 시 활용) |
| UR-11 | 보안 / 도난 감지 | 쑈삥끼가 지정된 마트 구역을 벗어나면 도난으로 판단하고 직원을 호출한다. 호출은 중앙 관제 대시보드 알람 및 로그, 쑈삥끼 LED 점멸, LCD 화면 표시로 이루어진다. |
| UR-12 | 쇼핑 리스트 / 추가 | 주인은 웹앱에서 "물건 추가" 모드를 선택한 후 쑈삥끼의 카메라에 상품 QR 코드를 보여주어 장바구니에 물건을 추가할 수 있다. |
| UR-13 | 쇼핑 리스트 / 삭제 | 주인은 웹앱의 장바구니 목록에서 항목 옆 삭제 버튼을 눌러 물건을 제거할 수 있다. |
| UR-14 | 결제 | 쑈삥끼가 결제 구역에 진입하면 장바구니의 물건들이 자동으로 결제 처리된다. (사용자의 카드 정보를 등록하며, 실제 결제 API 연동 없이 데모용 가상 결제로 동작한다.) 결제 오류 발생 시 직원을 호출하고 앱과 LCD에 오류를 표시한다. |
| UR-15 | 물건 찾기 | 주인은 웹앱에서 텍스트 입력 또는 음성-텍스트 변환(STT)을 통해 찾는 물건의 위치를 쑈삥끼에게 질의할 수 있다. |
| UR-16 | 물건 안내 | 쑈삥끼는 물건이 진열된 위치로 이동하여 주인에게 위치를 안내한 뒤, 자동으로 주인 추적을 재개한다. 안내 이동 중에는 LED가 노란색으로 표시되며, 도착 시 웹앱 알림과 LED로 주인에게 알린다. |
| UR-17 | 귀환 | 주인이 웹앱에서 "보내주기"를 선택하면 쑈삥끼는 카트 대기 구역으로 자동 복귀한다. 복귀 완료 시 세션이 종료되며, 포즈 정보가 삭제되고 로그인 상태가 초기화된다. 장바구니가 비어 있지 않으면 앱에서 알림을 표시하고 실행되지 않는다. |
| UR-18 | 배터리 / 직원 호출 | 쑈삥끼의 배터리가 일정 수준 이하로 떨어지면 직원을 호출한다. 호출은 중앙 관제 대시보드 알람 및 로그, 쑈삥끼 LED 점멸, LCD 화면 표시로 이루어진다. |
| UR-19 | 중앙 관제 대시보드 | 직원은 중앙 관제 대시보드에서 마트 맵 위에 모든 쑈삥끼의 실시간 위치와 현재 동작 모드를 한눈에 확인할 수 있다. |
| UR-20 | 중앙 관제 대시보드 | 직원을 호출하는 이벤트(장시간 대기, 도난 감지, 배터리 부족, 결제 오류)가 발생하면 중앙 관제 대시보드에 알람이 표시되고 로그로 기록된다. |
| UR-21 | 세션 / 사용 중 차단 | 이미 사용 중인 쑈삥끼의 QR 코드를 다른 사람이 스캔하면 "현재 사용 중입니다" 메시지를 표시하고 접속을 차단한다. |
| UR-22 | 대기 / 통행 방해 회피 | 대기 중 쑈삥끼가 통행을 방해하는 위치에 있을 경우, 주변 통행자를 감지하여 자동으로 비켜준다. |
| UR-23 | 알람 / 해제 | 알람은 두 가지 방법으로 해제할 수 있다. ① 중앙 관제 대시보드에서 직원이 직접 해제. ② 직원이 쑈삥끼에 직접 방문하여 웹앱에서 4자리 비밀번호를 입력해 해제. 도난 알람 해제 시 세션이 강제 종료되며 로봇은 등록 대기 상태로 초기화된다. 그 외 알람(배터리 부족·장시간 대기·결제 오류) 해제 시 대기 모드로 복귀한다. |

## 세션 흐름

```
[QR 스캔]
    ↓
[로봇 사용 중?] ── Yes ──▶ ["현재 사용 중입니다" 표시 후 차단]
    │ No
    ↓
[세션 유효?] ──── Yes ──▶ [메인화면]
    │ No
    ↓
[로그인 / 회원가입]
    ↓
[포즈 스캔 (매 사용마다)]
    ↓
[메인화면 — 쇼핑 보조]
    ↓
["보내주기"]
    ↓
[로봇 귀환] ──▶ [세션 종료: 로그아웃 + 포즈 데이터 삭제]
```

## 카테고리 정의

| 카테고리 | 설명 |
|---|---|
| 진입 | QR 스캔으로 웹앱 접속 |
| 계정 | 회원가입 / 로그인 (계정 정보 영구 보관) |
| 계정 / 포즈 스캔 | 매 사용 시 외형 캘리브레이션 |
| 세션 | 세션 유지 및 종료 흐름 |
| 세션 / 사용 중 차단 | 이미 사용 중인 로봇 QR 접속 차단 |
| 추적 | 주인 인식 후 마트 내 추종 |
| 추적 / 재탐색 | 주인 소실 시 제자리 회전 탐색 및 대기 전환 |
| 대기 / 직원 호출 | 대기 모드 진입 및 비상 상황 처리 |
| 앱 / 상태 확인 | 웹앱 UI — 지도, 위치, 현재 모드 표시 |
| LCD / 상태 표시 | LCD — 현재 모드 + 웹앱 QR 코드 상시 표시 |
| 앱 / 모드 전환 | 웹앱 UI — 로봇 동작 모드 원격 제어 |
| 보안 / 도난 감지 | 구역 이탈 감지 및 알림 |
| 쇼핑 리스트 / 추가 | "물건 추가" 모드에서 QR 코드 스캔 |
| 쇼핑 리스트 / 삭제 | 웹앱에서 장바구니 항목 삭제 |
| 결제 | 결제 구역 자동 정산 (데모용 가상 결제) |
| 물건 찾기 | 텍스트 또는 STT로 아이템 위치 질의 |
| 물건 안내 | 아이템 위치로 로봇 이동 안내 후 추적 재개 |
| 귀환 | 사용 종료("보내주기") 후 자동 복귀 및 세션 종료 |
| 배터리 / 직원 호출 | 배터리 상태 모니터링 및 알림 |
| 중앙 관제 대시보드 | 직원용 — 전체 로봇 위치/상태 맵 표시 + 알람/로그 |
| 대기 / 통행 방해 회피 | 대기 중 통행자 감지 시 자동 회피 이동 |
| 알람 / 해제 | 관제 대시보드 직접 해제 또는 웹앱 4자리 비밀번호 입력으로 해제 |

## 동작 모드 정의

쑈삥끼의 동작 모드는 앱과 LCD 양쪽에 표시된다.

| 모드 | 전환 조건 |
|---|---|
| 등록 대기 | 포즈 스캔 전 초기 상태 |
| 추적 중 | 포즈 스캔 완료 후, 주인을 인식한 상태 |
| 재탐색 중 | 주인을 놓쳤을 때, 제자리 회전으로 탐색 중 |
| 대기 중 | 재탐색 실패 또는 앱에서 수동 전환 |
| 물건 추가 중 | "물건 추가" 모드 진입, QR 스캔 대기 중 |
| 안내 중 | 물건 진열대로 이동 중 |
| 귀환 중 | "보내주기" 실행 후 대기 구역으로 복귀 중 |
| 알람 | 도난 감지 / 배터리 부족 / 장시간 대기 / 결제 오류 |
