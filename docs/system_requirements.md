# 시스템 요구사항 (System Requirements)

> **프로젝트:** 쑈삥끼 (ShopPinkki)
> **팀:** 삥끼랩 | 에드인에듀 자율주행 프로젝트 2팀

시스템 요구사항은 사용자 요구사항(UR)을 시스템이 어떻게 구현하는지를 정의합니다.

---

## SR 테이블

### 하드웨어 / 플랫폼

| SR ID | 연관 UR | Description |
|---|---|---|
| SR-01 | 전체 | 로봇은 Raspberry Pi 5 (8GB) 기반 Pinky Pro를 사용한다. |
| SR-02 | UR-09 | 쑈삥끼 LCD(ST7789, 2.4인치)는 터치 입력을 지원하지 않는다. 모든 사용자 조작은 웹앱을 통해 이루어진다. |

### 네트워크

| SR ID | 연관 UR | Description |
|---|---|---|
| SR-03 | 전체 | 모든 Pinky와 중앙 관제 노트북은 동일한 WiFi 네트워크(LAN)에 연결되어야 한다. (공유기, 핫스팟 등 수단 무관) |
| SR-04 | 전체 | 각 Pinky는 고유한 고정 IP를 가져야 한다. 공유기의 MAC 주소 기반 IP 예약(DHCP 고정 할당)으로 설정한다. Pinky 번호를 IP 끝자리로 사용한다. (Pinky #54 → `192.168.x.54`, Pinky #18 → `192.168.x.18`) |
| SR-05 | 전체 | Pinky는 AP 모드가 아닌 클라이언트(Station) 모드로 동작하며, 기존 WiFi 네트워크에 접속한다. |

### 계정 및 세션

| SR ID | 연관 UR | Description |
|---|---|---|
| SR-10 | UR-02 | 사용자 계정은 ID, 비밀번호, 이름, 전화번호, 카드 정보로 구성되며 중앙 서버(노트북)에 영구 보관된다. Pi 5는 계정 DB를 보유하지 않으며, 로그인/회원가입 요청을 중앙 서버로 전달한다. |
| SR-11 | UR-02 | QR 스캔 후 최초 접속 시 회원가입 화면을 표시한다. 이미 계정이 있으면 로그인 화면을 표시한다. |
| SR-12 | UR-03 | 로그인/회원가입 완료 후 매 사용마다 포즈 스캔(외형 캘리브레이션)을 수행한다. |
| SR-13 | UR-03 | 포즈 스캔은 전면→우측→후면→좌측 순서로 4방향을 촬영한다. 각 포즈 완료 시 부저로 알림을 제공한다. |
| SR-14 | UR-03 | 포즈 스캔 중 LCD에는 Pi 5 카메라로 보이는 실시간 영상을 표시한다. 웹앱에서는 LCD를 보도록 안내 문구를 표시한다. |
| SR-15 | UR-04 | 세션 쿠키가 유효한 경우 QR 재접속 시 로그인 및 포즈 스캔을 건너뛰고 메인화면으로 이동한다. |
| SR-16 | UR-04 | 세션 쿠키가 만료된 경우 ID + 비밀번호 로그인 후 포즈 스캔을 수행한다. |
| SR-17 | UR-17 | "보내주기" 완료 시 Pi 5의 세션을 종료한다. 포즈 데이터를 삭제하고 로그인 상태를 초기화한다. |
| SR-18 | UR-21 | Pi 5는 활성 세션 보유 여부를 상태로 관리한다. QR 접속 시 활성 세션이 있으면 접속을 차단하고 "현재 사용 중" 페이지를 반환한다. |

### 주인 인식

| SR ID | 연관 UR | Description |
|---|---|---|
| SR-20 | UR-05 | 주인 인식은 Pi 5에서 로컬로 처리한다. 외부 서버 오프로딩 없음. |
| SR-21 | UR-05 | 주인 탐지는 YOLOv8n 모델을 사용한다. 성능 최적화가 필요한 경우 NCNN 변환을 적용한다. |
| SR-22 | UR-05 | 주인 재식별(ReID)은 HSV 색상 히스토그램(상/하의 분리) 기반 특징 비교로 수행한다. |

### 주행 / 네비게이션

| SR ID | 연관 UR | Description |
|---|---|---|
| SR-30 | UR-05, UR-06 | 주인 추적 주행은 인식된 주인의 위치 좌표를 기반으로 비례 제어(P-Control)를 사용한다. |
| SR-31 | UR-05 | 주인 추적 중 장애물 회피는 RPLiDAR C1 스캔 데이터를 모니터링하여 전방 일정 거리 이내에 장애물 감지 시 속도를 감소하거나 정지하는 반응형 회피 레이어로 처리한다. P-Control과 병렬 동작하며 /cmd_vel 충돌 없이 출력을 보정한다. |
| SR-32 | UR-11 | 구역 이탈 감지(도난 방지)는 AMCL pose의 x, y 좌표 임계값으로 판단한다. (예: 맵 경계 좌표 초과 시 이탈로 판정) |
| SR-33 | UR-14 | 결제 구역 진입 감지는 AMCL pose의 x, y 좌표 임계값으로 판단한다. 결제 구역은 맵상에 사전 정의된 직사각형 영역이다. |
| SR-34 | UR-14 | 결제 오류 발생 시 중앙 관제 대시보드 알람 및 로그, LED 점멸, LCD 화면 표시, 앱 알림으로 직원을 호출한다. |
| SR-35 | UR-16, UR-17 | 물건 안내 및 귀환 주행은 Nav2 Waypoint Navigation을 사용한다. |
| SR-36 | UR-22 | 대기 중 RPLiDAR C1으로 근접 통행자를 감지하면 Nav2를 통해 옆으로 소폭 이동하여 통행로를 확보한다. 이동 후 다시 정지 대기한다. |
| SR-37 | UR-06 | 재탐색 중 동작은 제자리에서 일정 각도씩 회전하며 YOLOv8n으로 주인을 탐색한다. 360° 전방향 탐색 후에도 주인을 찾지 못하면 대기 모드로 전환한다. |

### 쇼핑 리스트 / QR 스캔

| SR ID | 연관 UR | Description |
|---|---|---|
| SR-40 | UR-12 | 상품 QR 코드 스캔은 OpenCV `QRCodeDetector`를 사용한다. 별도 라이브러리 불필요. |
| SR-41 | UR-12 | 상품 QR 코드에는 상품명과 가격 정보가 인코딩된다. |

### 웹앱 (주인용 — Pi 5 호스팅)

| SR ID | 연관 UR | Description |
|---|---|---|
| SR-50 | 전체 | 주인용 웹앱은 Pi 5에서 Flask로 호스팅한다. |
| SR-51 | UR-01 | LCD에 표시되는 QR 코드는 Pi 5의 웹앱 주소(`http://<Pi_IP>:<PORT>`)를 인코딩한다. Pi 5는 시작 시 자신의 현재 IP를 동적으로 읽어 QR 코드를 생성한다. (하드코딩 금지) |
| SR-52 | UR-08 | 주인용 웹앱은 Pi 5와 WebSocket으로 연결하여 로봇 상태(위치, 모드, 배터리 잔량)를 실시간으로 수신하고, 앱에서의 모드 전환 명령도 동일 연결로 전송한다. |
| SR-53 | UR-14 | 카드 정보는 최초 회원가입 시 등록한다. (SR-10 참조) |
| SR-54 | UR-15 | 웹앱의 물건 찾기 STT 기능은 브라우저 Web Speech API를 사용한다. 별도 서버 처리 불필요. |

### 중앙 관제 서버 (노트북 호스팅)

| SR ID | 연관 UR | Description |
|---|---|---|
| SR-60 | UR-19, UR-20 | 관제 대시보드는 노트북에서 호스팅하며, 직원이 브라우저로 접속한다. |
| SR-61 | UR-19 | 각 Pi 5는 중앙 서버와 WebSocket 연결을 유지하며 자신의 위치, 동작 모드, 배터리 잔량을 1~2Hz 주기로 실시간 전송한다. 연결 끊김 시 자동 재연결을 시도한다. (ROS 토픽 구독 방식 미사용 — DDS 오버헤드 방지) |
| SR-62 | UR-19 | 관제 대시보드는 마트 맵 이미지 위에 각 로봇의 실시간 위치를 오버레이하여 표시한다. |
| SR-63 | UR-20 | 이벤트(장시간 대기, 도난 감지, 배터리 부족, 결제 오류) 발생 시 기존 WebSocket 연결을 통해 중앙 서버로 알람을 즉시 전송하고 로그에 기록한다. |
| SR-64 | UR-23 | 알람 해제 경로는 두 가지다. ① 중앙 관제 대시보드: 별도 인증 없이 해제 버튼으로 즉시 해제. 중앙 서버가 해당 Pi 5에 해제 명령을 WebSocket으로 전송한다. ② 웹앱(현장): 직원이 쑈삥끼에 직접 방문하여 웹앱의 알람 해제 화면에서 4자리 비밀번호를 입력한다. Pi 5가 비밀번호를 검증 후 자체적으로 알람을 해제한다. 두 경로 모두 도난 알람은 세션 강제 종료 및 등록 대기 초기화, 그 외 알람은 대기 모드 복귀로 처리한다. |
